# PRODUCT.md

## Register
brand

## Product
Landing page for Vallum, a Rust CLI security proxy that sits between AI coding
agents (Claude Code, Cursor, Codex, Gemini CLI) and the shell. It sanitizes
command output before the model sees it: secret redaction, prompt-injection
neutralization, context-preserving truncation, untrusted-output wrapping,
pre-execution guardrails, audit logging.

## Users
Developers who run AI coding agents daily; security-conscious teams running
agents in CI/CD. Technical buyers who distrust marketing fluff and respond to
real commands, real metrics, real terminal output.

## Purpose
Convince a developer in one scroll that terminal output is untrusted input,
show the product working (before/after terminal demo), get them to run one
install command.

## Brand personality
Monumental, defensive, precise. Roman frontier engineering applied to a
modern terminal. Confidence through evidence (published eval metrics), not
adjectives.

## References
Trajan-inscription epigraphy meets a modern terminal emulator. Dark
basalt-and-bronze; discipline of Linear, texture of a stone monument.

## Anti-references
AI-purple SaaS gradients, glassmorphism, neon glow, generic three-card
feature grids, marketing verbs ("seamless", "unleash"). No light mode: the
page is a night watch on a wall.

## Accessibility
WCAG AA contrast minimum on all text; full `prefers-reduced-motion`
fallbacks; keyboard-reachable tabs and copy buttons.

## Strategic principles
1. The boundary is the story: every visual motif derives from the wall
   (crenellation lines, gates, the inscription).
2. Show, don't claim: the hero demo is real product output; metrics come
   from the committed eval corpus.
3. One accent (bronze #c9a24d) carries the entire identity.
