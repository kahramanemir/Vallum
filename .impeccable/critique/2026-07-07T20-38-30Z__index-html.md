## Priority Issues

- [P0] Reveal animations silently drop the page's most persuasive sections (pipeline, guardrail, METRICS, motto) on ordinary fast scroll — reproduced 2 ways in real Chrome (fast wheel jump; full-page screenshot with no scroll). `.reveal` rests at opacity:0 until IntersectionObserver fires (script.js:166-185, style.css:991-1001); fast scroll skips it. Hits the credibility climax ("Measured, not promised") hardest; also blanks headless/OG renders. Fix: never rest at opacity 0 (animate transform only), or idle-sweep force-add `.in` to in-viewport reveals, or animation-timeline: view().
- [P1] Install section's default tab is `curl … | sh` — the exact command shape the page's own guardrail demo flags as "ask" two sections earlier (index.html:170-171 vs 231-232). Self-undermining at the conversion moment for a fluff-distrusting audience. Fix: lead with brew/cargo tab, or put checksum/attestation verification inline next to the curl command.
- [P2] Mobile experience: hero clean-terminal payoff blank for first ~4-5s (choreography too slow for mobile attention; verified 390px screenshot); .nav-links display:none below 768px with no replacement (no jump to Install). Fix: shorten/skip hero choreography on small viewports; add minimal mobile nav or floating install anchor.
- [P2] Two of four headline metrics duplicated verbatim (1.000 recall + 80.8% appear in threat cards AND metrics row) — dilutes the "Measured, not promised" reveal. Fix: show injection precision/recall in threat cards instead, keep metrics row as the single reveal.
- [P3] Contrast batch: --text3 #837c6f computes 4.06-4.4:1 on --bg1/--bg2 (inactive tabs, pane titles, marquee, copy-btn) vs DESIGN.md's own AA commitment. Fix: lighten --text3 or add --text3-onpanel.
- [P3] Note line-length ~90ch on .pipeline-note/.install-note (cap ~75ch).
